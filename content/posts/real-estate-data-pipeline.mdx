부동산 데이터는 국토부, 행안부, 지자체 등 여러 기관에 분산되어 있다. 각 기관마다 API 형식, 인코딩, 업데이트 주기가 제각각이다.
이 데이터를 통합해서 AVM과 서비스에 공급하는 파이프라인을 구축한 과정을 정리한다.

## 데이터 소스 현황

우리가 수집하는 주요 데이터 소스:

| 소스 | 기관 | 업데이트 | 형식 | 주요 데이터 |
|------|------|---------|------|------------|
| 실거래가 | 국토부 | 월 1회 | XML API | 매매/전월세 거래 |
| 건축물대장 | 국토부 | 수시 | REST API | 건물 정보, 면적 |
| 공시지가 | 국토부 | 연 1회 | CSV | 토지 가격 |
| 행정동 경계 | 행안부 | 연 1회 | SHP | GIS 경계 |
| 지하철 노선 | 국토교통부 | 수시 | REST API | 역 좌표 |

## 아키텍처 설계

```
[공공 API] → [Collector] → [S3 Raw] → [Transformer] → [PostgreSQL] → [API Server]
                                              ↕
                                        [Airflow DAG]
```

**설계 원칙:**
1. Raw 데이터는 S3에 원본 그대로 보관 (재처리 가능성)
2. 변환 로직은 멱등성 보장 (같은 입력 → 같은 출력)
3. 수집 실패는 재시도 가능하도록 체크포인트 저장

## 수집기(Collector) 구현

국토부 API는 공공데이터포털을 통해 제공되는데, 요청 방식이 REST가 아닌 SOAP에 가깝고 응답이 XML이다.

```python
import httpx
import xmltodict
from typing import AsyncIterator
import asyncio

class MolitTransactionCollector:
    BASE_URL = "https://apis.data.go.kr/1613000/RTMSDataSvcAptTradeDev/getRTMSDataSvcAptTradeDev"

    def __init__(self, api_key: str):
        self.api_key = api_key
        self.client = httpx.AsyncClient(timeout=30.0)

    async def fetch_month(self, region_code: str, year_month: str) -> list[dict]:
        """특정 지역/월의 거래 데이터를 페이지네이션하여 전부 수집."""
        all_items = []
        page = 1

        while True:
            params = {
                "serviceKey": self.api_key,
                "LAWD_CD": region_code,
                "DEAL_YMD": year_month,
                "pageNo": page,
                "numOfRows": 1000,
            }
            resp = await self.client.get(self.BASE_URL, params=params)
            resp.raise_for_status()

            data = xmltodict.parse(resp.text)
            items = data["response"]["body"].get("items", {})
            if not items:
                break

            item_list = items.get("item", [])
            if isinstance(item_list, dict):
                item_list = [item_list]  # 단건 응답 처리

            all_items.extend(item_list)

            total_count = int(data["response"]["body"]["totalCount"])
            if len(all_items) >= total_count:
                break
            page += 1

        return all_items
```

<Callout type="info">
국토부 API는 응답 건수가 1건일 때 list가 아닌 dict로 반환한다. 이 엣지 케이스를 놓치면 파이프라인이 조용히 실패한다.
</Callout>

## S3 파티셔닝 전략

Raw 데이터는 Hive 스타일 파티션으로 저장한다. 나중에 Athena로 직접 쿼리가 가능하고, Transformer가 날짜 범위로 효율적으로 읽을 수 있다.

```
s3://proptech-datalake/
└── raw/
    └── transactions/
        └── year=2026/
            └── month=01/
                └── region=11110/
                    └── 20260102T143022.json.gz
```

```python
import boto3
import gzip
import json
from datetime import datetime

def upload_raw(items: list[dict], region: str, year_month: str) -> str:
    s3 = boto3.client("s3")
    year, month = year_month[:4], year_month[4:]
    timestamp = datetime.utcnow().strftime("%Y%m%dT%H%M%S")

    key = f"raw/transactions/year={year}/month={month}/region={region}/{timestamp}.json.gz"

    body = gzip.compress(json.dumps(items, ensure_ascii=False).encode("utf-8"))
    s3.put_object(
        Bucket="proptech-datalake",
        Key=key,
        Body=body,
        ContentEncoding="gzip",
        ContentType="application/json",
    )
    return key
```

## 변환기(Transformer) — 주소 정규화

가장 복잡한 부분이다. 국토부 데이터의 주소 필드는 파편화되어 있다.

```python
from dataclasses import dataclass
import re

@dataclass
class NormalizedAddress:
    sido: str
    sigungu: str
    eupmyeondong: str
    jibun: str
    road_name: str | None
    road_number: str | None
    latitude: float | None
    longitude: float | None

def normalize_transaction(raw: dict) -> dict:
    """국토부 실거래가 원본 레코드를 정규화."""
    # 면적: 문자열 "84.78" → float
    area = float(raw.get("excluUseAr", 0) or 0)

    # 금액: "85,000" (만원) → int (원)
    price_str = raw.get("dealAmount", "0").replace(",", "").strip()
    price = int(price_str) * 10_000

    # 층수: "  3" 같은 공백 포함
    floor_str = raw.get("floor", "0").strip()
    floor = int(floor_str) if floor_str.lstrip("-").isdigit() else 0

    # 건축연도: 실거래 데이터에는 없음, 건축물대장과 조인 필요
    return {
        "area_exclusive": area,
        "price": price,
        "floor": floor,
        "deal_year": int(raw.get("dealYear", 0)),
        "deal_month": int(raw.get("dealMonth", 0)),
        "deal_day": int(raw.get("dealDay", 0)),
        "sigungu_code": raw.get("regionalCode", ""),
        "dong": raw.get("umdNm", ""),
        "jibun": raw.get("jibun", ""),
        "apt_name": raw.get("aptNm", ""),
    }
```

## Airflow DAG 구성

월별 배치와 실시간 갱신을 분리했다.

```python
from airflow import DAG
from airflow.operators.python import PythonOperator
from airflow.utils.dates import days_ago
from datetime import timedelta

with DAG(
    dag_id="real_estate_monthly_pipeline",
    schedule_interval="0 2 5 * *",  # 매월 5일 오전 2시 (전월 데이터 확정 후)
    start_date=days_ago(1),
    catchup=False,
    default_args={
        "retries": 3,
        "retry_delay": timedelta(minutes=10),
        "on_failure_callback": notify_slack,
    },
) as dag:

    collect = PythonOperator(
        task_id="collect_transactions",
        python_callable=run_collector,
        op_kwargs={"region_codes": ALL_REGION_CODES},
    )

    transform = PythonOperator(
        task_id="transform_and_load",
        python_callable=run_transformer,
    )

    refresh_model = PythonOperator(
        task_id="trigger_model_retrain",
        python_callable=trigger_avm_retrain,
    )

    collect >> transform >> refresh_model
```

<Callout type="warning">
공공 API는 월말~월초에 집중적으로 데이터가 갱신된다. DAG 실행 시간을 매월 5일로 잡으면 대부분의 전월 데이터가 확정된 후 수집할 수 있다.
</Callout>

## 성능 최적화

초기 구현은 지역 코드를 순차 처리했는데, 전국 처리에 4시간이 걸렸다.

```python
import asyncio

async def collect_all_regions(region_codes: list[str], year_month: str) -> None:
    """세마포어로 동시성 제한 — API 과부하 방지."""
    semaphore = asyncio.Semaphore(10)  # 동시 요청 최대 10개

    async def collect_one(code: str) -> None:
        async with semaphore:
            collector = MolitTransactionCollector(api_key=API_KEY)
            items = await collector.fetch_month(code, year_month)
            if items:
                upload_raw(items, code, year_month)

    await asyncio.gather(*[collect_one(code) for code in region_codes])

# 4시간 → 18분으로 단축
asyncio.run(collect_all_regions(ALL_REGION_CODES, "202601"))
```

## 현재 상태

현재 파이프라인이 처리하는 데이터 규모:
- 누적 거래 데이터: **약 3,200만 건** (2006년~현재)
- 월별 신규 유입: **약 8만 건**
- 처리 시간: **전국 기준 18분** (비동기 처리)
- DB 크기: PostgreSQL 기준 **약 45GB**

<Callout type="tip">
부동산 데이터 파이프라인의 병목은 API 속도 제한이다. 공공데이터포털 API 키는 일별 호출 제한이 있으므로, 프로덕션에서는 키를 복수로 운영하거나 데이터를 로컬 캐시에 저장해두는 전략이 필요하다.
</Callout>
